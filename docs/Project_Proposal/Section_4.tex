\subsection{Development Tools and Process}

The team plans to follow an iterative and agile-inspired development process, emphasizing early prototypes, incremental feature development, and frequent testing. Rather than attempting a single fully featured implementation, the project will progress through multiple functional prototypes, each adding capability and validation.

Collaboration and project management tools include:
\begin{itemize}
	\item \textbf{Version Control:} Bitbucket (schematics, PCB layouts, firmware documentation)
	\item \textbf{Task Management:} Jiro for sprint planning and tracking
	\item \textbf{Communication:} Discord/Slack for daily communication
	\item \textbf{Documentation:} Confluence/Google Drive for shared documents
\end{itemize}

From a technical perspective, development tools are expected to include:
\begin{itemize}
	\item \textbf{PCB Design:} KiCad
	\item \textbf{Circuit Simulator:} LTspice or KiCad simulation tools
	\item \textbf{Firmware Development:} Arduino IDE or PlatformIO with VSCode
	\item \textbf{Programming Languages:} C/C++ for embedded firmware, Python for test scripts
	\item \textbf{CAN Bus Analysis:} Saleae Logic 2, Sigrok Pulseview
	\item \textbf{3D Modeling CAD:} Fusion 360, SolidWorks, Onshape
	\item \textbf{Documentation:} LaTeX, Microsoft Word, Google Docs
\end{itemize}
